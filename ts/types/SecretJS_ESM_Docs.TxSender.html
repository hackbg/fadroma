    • exec: SingleMsgTx<Tx.MsgExecParams>

      MsgExec attempts to execute the provided messages using authorizations granted to the grantee. Each message should have only one signer corresponding to the granter of the authorization.

    • grant: SingleMsgTx<Tx.MsgGrantParams>

      MsgGrant is a request type for Grant method. It declares authorization to the grantee on behalf of the granter with the provided expiration time.

    • revoke: SingleMsgTx<Tx.MsgRevokeParams>

      MsgRevoke revokes any authorization with the provided sdk.Msg type on the granter's account with that has been granted to the grantee.

  • broadcast: ((messages, txOptions?) => Promise<TxResponse>)

    Sign and broadcast a transaction to Secret Network.

    Param: options

    Options for signing and broadcasting

    Param: options.gasLimit

    Param: options.gasPriceInFeeDenom

    E.g. gasPriceInFeeDenom=0.1 & feeDenom="uscrt" => Total fee for tx is 0.1 * gasLimituscrt.

    Param: options.feeDenom

    Param: options.memo

    Param: options.waitForCommit

    If false returns immediately with transactionHash. Defaults to true.

    Param: options.broadcastTimeoutMs

    How much time (in milliseconds) to wait for tx to commit on-chain. Ignored if waitForCommit = false.

    Param: options.broadcastCheckIntervalMs

    When waiting for the tx to commit on-chain, how much time (in milliseconds) to wait between checks. Smaller intervals will cause more load on your node provider. Keep in mind that blocks on Secret Network take about 6 seconds to finalize. Ignored if waitForCommit = false.

    Param: options.broadcastMode

    If BroadcastMode.Sync - Broadcast transaction to mempool and wait for CheckTx response.

    See

    • https://docs.tendermint.com/master/rpc/#/Tx/broadcast_tx_sync. If BroadcastMode.Async Broadcast transaction to mempool and do not wait for CheckTx response.
    • https://docs.tendermint.com/master/rpc/#/Tx/broadcast_tx_async.

    Param: options.explicitSignerData

    explicitSignerData can be used to override chainId, accountNumber & accountSequence. This is useful when using BroadcastMode.Async or when you don't want secretjs to query for accountNumber & accountSequence from the chain. (smoother in UIs, less load on your node provider).

    Param: options.explicitSignerData.accountNumber

    Param: options.explicitSignerData.sequence

    Param: options.explicitSignerData.chainId

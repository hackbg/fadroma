  • broadcast: ((messages: SecretJS.Msg[], txOptions?: SecretJS.TxOptions) => Promise<SecretJS.TxResponse>)
      • (messages: SecretJS.Msg[], txOptions?: SecretJS.TxOptions): Promise<SecretJS.TxResponse>

      • Sign and broadcast a transaction to Secret Network.

        See

        • https://docs.tendermint.com/master/rpc/#/Tx/broadcast_tx_sync. If Async Broadcast transaction to mempool and do not wait for CheckTx response.
        • https://docs.tendermint.com/master/rpc/#/Tx/broadcast_tx_async.

        Parameters

        • messages: SecretJS.Msg[]

          A list of messages, executed sequentially. If all messages succeeds then the transaction succeed, and the resulting TxResponse object will have code = 0. If at lease one message fails, the entire transaction is reverted and TxResponse code field will not be 0.

      • (messages: SecretJS.Msg[], txOptions?: SecretJS.TxOptions): Promise<SimulateResponse>

      • Simulates a transaction on the node without broadcasting it to the chain. Can be used to get a gas estimation or to see the output without actually committing a transaction on-chain. The input should be exactly how you'd use it in broadcast.

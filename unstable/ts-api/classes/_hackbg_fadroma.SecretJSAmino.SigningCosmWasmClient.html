constructor

  • new SigningCosmWasmClient(apiUrl: string, senderAddress: string, signer: OfflineSigner | SecretJSAmino.SigningCallback, seedOrEnigmaUtils?: Uint8Array | SecretUtils, customFees?: Partial<SecretJSAmino.FeeTable>, broadcastMode?: SecretJSAmino.BroadcastMode): SecretJSAmino.SigningCosmWasmClient

  • Overrides CosmWasmClient.constructor

    • Defined in node_modules/.pnpm/secretjs@0.17.5/node_modules/secretjs/types/signingcosmwasmclient.d.ts:68

    Creates a new client with signing capability to interact with a CosmWasm blockchain. This is the bigger brother of CosmWasmClient.

    This instance does a lot of caching. In order to benefit from that you should try to use one instance for the lifetime of your application. When switching backends, a new instance must be created.

    Parameters

    • apiUrl: string

      The URL of a Cosmos SDK light client daemon API (sometimes called REST server or REST API)

    • senderAddress: string

      The address that will sign and send transactions using this instance

    • signer: OfflineSigner | SecretJSAmino.SigningCallback

      An asynchronous callback to create a signature for a given transaction. This can be implemented using secure key stores that require user interaction. Or a newer OfflineSigner type that handles that stuff

    • Optional seedOrEnigmaUtils: Uint8Array | SecretUtils
    • Optional customFees: Partial<SecretJSAmino.FeeTable>

      The fees that are paid for transactions

    • Optional broadcastMode: SecretJSAmino.BroadcastMode

      Defines at which point of the transaction processing the postTx method (i.e. transaction broadcasting) returns

    Returns SecretJSAmino.SigningCosmWasmClient
